## 와이어프레임 개요

### 목적
- 개발 전 화면 구조 합의
- 정보 계층 구조(Information Hierarchy) 검증
- 반응형 레이아웃 설계
- 개발팀과의 원활한 커뮤니케이션

### 표기 규칙

```
[Component Name]
  ├─ Element 1
  ├─ Element 2
  └─ Element 3

● 필수 요소
○ 선택 요소
⚡ 인터랙티브 요소
🔒 권한 필요
```

## 디자인 시스템

### 레이아웃 그리드
- **컨테이너 최대 너비**: 1440px
- **컬럼 수**: 12 columns
- **거터(Gutter)**: 16px
- **마진**: 24px (모바일), 48px (데스크톱)

### 스페이싱 시스템
| Token | Value | 사용 예시 |
|-------|-------|-----------|
| `--space-xs` | 4px | 아이콘 여백 |
| `--space-sm` | 8px | 텍스트 간격 |
| `--space-md` | 16px | 카드 내부 패딩 |
| `--space-lg` | 24px | 섹션 간 간격 |
| `--space-xl` | 32px | 메인 컨텐츠 여백 |
| `--space-2xl` | 48px | 대형 섹션 여백 |

### 컬러 팔레트 (Salesforce SLDS 기반)

#### Primary Colors
- **Salesforce Blue**: `#00A1E0` (Primary Action)
- **Dark Blue**: `#16325C` (Text, Headers)
- **Light Blue**: `#E6F4F9` (Background Highlight)

#### Neutral Colors
- **Gray-100**: `#F3F2F2` (Background)
- **Gray-300**: `#DDDBDA` (Border)
- **Gray-600**: `#706E6B` (Secondary Text)
- **Gray-900**: `#3E3E3C` (Primary Text)

#### Semantic Colors
- **Success**: `#04844B` (Green)
- **Error**: `#EA001E` (Red)
- **Warning**: `#FFB75D` (Orange)
- **Info**: `#00A1E0` (Blue)

### 타이포그래피
| 스타일 | 크기 | 두께 | 행간 | 사용처 |
|--------|------|------|------|--------|
| H1 | 32px | Bold | 1.2 | 페이지 제목 |
| H2 | 24px | Bold | 1.3 | 섹션 제목 |
| H3 | 20px | Semibold | 1.4 | 카드 제목 |
| Body | 16px | Regular | 1.5 | 본문 |
| Body-sm | 14px | Regular | 1.5 | 보조 텍스트 |
| Caption | 12px | Regular | 1.4 | 메타 정보 |
