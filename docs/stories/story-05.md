# US-05: 산학 협력 프로젝트 매칭 및 관리

## User Story
**As a** 기업 R&D 담당자 또는 대학 교수
**I want to** 산학 협력 프로젝트를 등록하고 적합한 파트너(학생/교수/기업)를 찾고
**So that** 실무 프로젝트를 통해 학생들에게 경험을 제공하고, 기업은 혁신적인 아이디어를 얻을 수 있다

## 시나리오

### Scenario 1: 기업이 산학 협력 프로젝트 등록
**Given** 기업 회원이 로그인한 상태에서
**When** "산학 협력" 메뉴에서 "프로젝트 등록" 버튼을 클릭하면
**Then** 다음 정보를 입력할 수 있는 폼이 표시된다:

**필수 항목:**
- 프로젝트명 (예: "AI 기반 고객 서비스 챗봇 개발")
- 프로젝트 유형:
  - 캡스톤 디자인 협업
  - 인턴십 프로젝트
  - 공동 연구개발 (R&D)
  - 해커톤/경진대회
- 참여 대상: 학부생, 대학원생, 교수
- 필요 인원: 1~10명 (슬라이더)
- 프로젝트 기간: 시작일 ~ 종료일
- 필요 기술 스택: React, Python, ML 등 (태그)
- 프로젝트 설명: 목표, 배경, 기대효과 (최대 2,000자)

**선택 항목:**
- 참여 혜택: 인턴십 전환 기회, 프로젝트 비용 지원, 수료증 발급
- 지원 예산: 범위 선택 (100만원 이하, 100~500만원, 500만원 이상)
- 협력 희망 대학: 특정 대학 지정 (미지정 시 전체 공개)

### Scenario 2: 교수가 기업 프로젝트 파트너 탐색
**Given** 대학 교수가 로그인한 상태에서
**When** "산학 협력 프로젝트" 검색 페이지에 접근하면
**Then** 다음 필터로 프로젝트를 검색할 수 있다:
- 프로젝트 유형
- 기술 분야: AI, 빅데이터, IoT, 웹/앱 개발 등
- 기업 규모: 대기업, 중견, 중소, 스타트업
- 지원 예산 범위
- 지역

**And** 검색 결과는 Card 형식으로 표시되며:
- 기업명 + 로고
- 프로젝트명
- 필요 기술 스택 (태그)
- 프로젝트 기간
- 모집 인원 / 현재 지원자 수
- 매칭 점수 (교수 연구 분야 기반)

### Scenario 3: 학생이 프로젝트 참여 신청
**Given** 학생이 관심 있는 프로젝트 상세 페이지를 보고 있을 때
**When** "참여 신청" 버튼을 클릭하면
**Then** 다음 정보를 입력할 수 있는 신청서가 표시된다:
- 지원 동기 (500자 이내)
- 관련 경험 (프로젝트, 수업, 자격증 등)
- 기여 가능 부분 (예: "React 프론트엔드 개발 담당 가능")
- 주당 투입 가능 시간: 10시간, 20시간, 30시간 이상
- 포트폴리오 링크 (선택)

**When** 신청서를 제출하면
**Then** 프로젝트 등록자(기업 또는 교수)에게 알림이 전송되고
**And** 학생의 "참여 프로젝트" 목록에 "신청 완료" 상태로 추가된다

### Scenario 4: 프로젝트 팀 구성 및 승인
**Given** 기업 담당자가 프로젝트 관리 페이지에 접근했을 때
**When** "신청자 목록" 탭을 클릭하면
**Then** 지원한 학생/교수 목록이 표시되고:
- 이름, 소속(학교/학과), 학년/직급
- 매칭 점수 (기술 스택 일치도)
- 신청 동기 및 경험 요약
- 프로필 보기 버튼

**And** 각 신청자에 대해 다음 액션 가능:
- "승인" → 팀 멤버로 추가
- "거절" → 거절 사유 선택 (선택사항)
- "보류" → 검토 중 상태 유지

**When** 승인을 누르면
**Then** 학생에게 승인 알림 전송 + 프로젝트 워크스페이스 접근 권한 부여

### Scenario 5: 프로젝트 워크스페이스
**Given** 프로젝트 팀이 구성된 후
**When** 팀 멤버(기업, 교수, 학생)가 프로젝트 페이지에 접근하면
**Then** 다음 기능이 제공되는 워크스페이스가 표시된다:

**Overview 탭:**
- 프로젝트 개요 및 목표
- 팀 멤버 목록 (역할 표시)
- 프로젝트 일정 (타임라인)
- 진행률 (0% → 100%)

**Tasks 탭:**
- 할 일 목록 (Kanban 보드)
- 담당자 배정, 마감일 설정
- 상태: To Do, In Progress, Done

**Files 탭:**
- 문서, 코드, 자료 업로드/다운로드
- 버전 관리 (Google Drive 또는 GitHub 연동)

**Communication 탭:**
- 팀 채팅 (실시간 메시징)
- 공지사항 게시판
- 회의록 작성 및 공유

### Scenario 6: 프로젝트 진행 상태 업데이트
**Given** 프로젝트 리더(기업 또는 교수)가 워크스페이스에 있을 때
**When** "진행 상태 업데이트" 버튼을 클릭하면
**Then** 다음을 입력할 수 있는 폼이 표시된다:
- 진행률: 슬라이더로 0~100% 설정
- 이번 주 활동 요약 (500자)
- 다음 주 계획
- 이슈 및 리스크 (선택사항)

**And** 저장 시 모든 팀 멤버에게 알림 전송

### Scenario 7: 프로젝트 완료 및 결과물 제출
**Given** 프로젝트가 종료일에 가까워졌을 때
**When** 리더가 "프로젝트 완료" 버튼을 클릭하면
**Then** 다음 정보를 입력할 수 있는 최종 보고서 폼이 표시된다:
- 프로젝트 성과 요약
- 최종 결과물 (파일 또는 링크)
- 팀원 평가 (각 멤버의 기여도 평가)
- 프로젝트 사진/동영상 (선택)

**When** 제출하면
**Then** 모든 팀 멤버에게 수료증이 자동 발급되고
**And** 학생의 프로필 "프로젝트 이력"에 자동 추가됨
**And** 기업은 우수 참여 학생을 "인재풀"에 저장 가능

### Scenario 8: 프로젝트 평가 및 피드백
**Given** 프로젝트가 완료된 후
**When** 각 팀 멤버에게 평가 요청 알림이 전송되면
**Then** 다음을 평가할 수 있는 폼이 제공된다:

**학생 → 기업/교수 평가:**
- 프로젝트 만족도: 별점 1~5
- 학습 효과: 별점 1~5
- 커뮤니케이션: 별점 1~5
- 코멘트 (선택)

**기업/교수 → 학생 평가:**
- 업무 수행 능력: 별점 1~5
- 협업 태도: 별점 1~5
- 기술 역량: 별점 1~5
- 추천 의향: "다른 기업에 추천하시겠습니까?" (예/아니오)

**And** 평가는 익명으로 처리되며, 평균 점수만 공개됨

### Scenario 9: 산학 협력 통계 대시보드 (대학 관리자용)
**Given** 대학 취업지원센터 담당자가 로그인했을 때
**When** "산학 협력 현황" 대시보드에 접근하면
**Then** 다음 통계가 시각화되어 표시된다:

**KPI Cards:**
- 진행 중인 프로젝트 수
- 참여 학생 수 (학과별)
- 협력 기업 수
- 평균 만족도

**차트:**
- 월별 프로젝트 추이 (라인 차트)
- 학과별 참여 현황 (바 차트)
- 기업 산업군별 분포 (파이 차트)
- 프로젝트 유형별 비율

## 인수 기준 (Acceptance Criteria)

### Functional
✅ 프로젝트 등록 시 필수 항목이 모두 입력되어야 함
✅ 학생은 동시에 최대 3개 프로젝트까지 참여 가능
✅ 프로젝트 승인 시 학생에게 30초 이내 알림 전송
✅ 워크스페이스 파일 업로드는 20MB 이하
✅ 채팅은 실시간으로 동기화됨 (WebSocket)
✅ 수료증은 PDF로 자동 생성되어 이메일 첨부

### Non-Functional
✅ 프로젝트 목록 로딩 시간 < 2초
✅ 채팅 메시지 전송 지연 < 1초
✅ 동시 50개 프로젝트 워크스페이스 운영 가능

### UI/UX (Salesforce SLDS 스타일)
✅ 프로젝트 카드는 Lightning Card 스타일
✅ 워크스페이스는 Tab 기반 레이아웃
✅ 팀 구성원은 Avatar + 역할 배지로 표시
✅ 진행률은 Progress Bar + 퍼센트로 시각화

## 의존성

## 기술적 고려사항

### Backend
- **API**: Project CRUD, Team Management, Task Management
- **Real-time**: Socket.io 또는 Firebase Realtime Database (채팅)
- **Database**:
  - Project 테이블: title, type, company_id, professor_id, status
  - ProjectMember 테이블: project_id, user_id, role, approved_at
  - ProjectTask 테이블: project_id, title, assignee, status, due_date
  - ProjectChat 테이블: project_id, sender_id, message, timestamp

### Frontend
- **Workspace**: Tab Component, Kanban Board (react-beautiful-dnd)
- **Chat**: Real-time Message Component
- **File Upload**: Drag & Drop Zone

### Integration
- **Storage**: AWS S3 (파일 저장)
- **PDF**: PDFKit (Node.js) 또는 Puppeteer (수료증 생성)
- **Notification**: SendGrid (프로젝트 알림)

## 테스트 시나리오

### 기능 테스트
1. 기업이 프로젝트 등록 → 데이터 정상 저장 확인
2. 학생이 프로젝트 신청 → 기업에게 알림 전송
3. 기업이 학생 승인 → 학생에게 워크스페이스 접근 권한 부여
4. 팀원이 채팅 메시지 전송 → 다른 멤버에게 실시간 표시
5. 프로젝트 완료 → 수료증 PDF 생성 및 이메일 전송

### 동시성 테스트
6. 50개 프로젝트 워크스페이스 동시 접속 → 응답 시간 측정
7. 채팅에 100개 메시지 동시 전송 → 순서 보장 확인

### 보안 테스트
8. 비멤버가 워크스페이스 접근 시도 → 403 Forbidden
9. 타인의 프로젝트 파일 다운로드 시도 → 권한 오류

## 참고사항
- MVP에서는 기본 워크스페이스만 제공, Phase 2에서 GitHub/Jira 연동
- 수료증 디자인은 기업별 커스터마이징 가능 (Phase 3)
- 프로젝트 우수 사례를 플랫폼 메인에 노출하여 참여 유도
