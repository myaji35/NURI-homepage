# US-04: 지원서 제출 및 채용 프로세스 추적

## User Story
**As a** 취업 준비 중인 대학생
**I want to** 관심 있는 채용공고에 지원서를 제출하고 진행 상태를 실시간으로 확인하고
**So that** 여러 기업에 효율적으로 지원하고, 각 채용 절차를 놓치지 않을 수 있다

## 시나리오

### Scenario 1: 채용공고 검색 및 상세 조회
**Given** 학생이 로그인한 상태에서
**When** 상단 검색바에 키워드(예: "프론트엔드 개발자")를 입력하면
**Then** 다음 필터와 함께 검색 결과가 표시된다:
- 고용 형태: 정규직, 인턴, 계약직
- 경력: 신입, 경력 1년, 경력 3년 이상
- 근무지: 지역 선택
- 기업 규모: 대기업, 중견, 중소, 스타트업
- 정렬: 최신순, 마감임박순, 매칭도순

**And** 각 공고 카드에는 다음 정보가 표시된다:
- 기업명 + 로고
- 공고 제목
- 주요 기술 스택 (태그)
- 근무지, 경력 요건
- 마감일 (D-7 형식)
- 매칭 점수 (로그인 시)

### Scenario 2: 공고 상세 페이지 및 즐겨찾기
**Given** 검색 결과에서 특정 공고를 클릭했을 때
**Then** 상세 페이지가 표시되고 다음 정보를 확인할 수 있다:
- 회사 소개
- 채용 분야 및 담당 업무
- 자격 요건 (필수/우대)
- 근무 조건 (근무지, 급여, 복리후생)
- 채용 절차 (서류 → 1차 면접 → 2차 면접 → 최종합격)
- 접수 기간 및 마감일

**And** 우측 상단에 다음 버튼이 표시된다:
- "⭐ 즐겨찾기" (토글 버튼)
- "지원하기" (Primary Action Button)

**When** 즐겨찾기 버튼을 클릭하면
**Then** 마이페이지의 "관심 공고" 목록에 추가된다

### Scenario 3: 지원서 작성 - 기본 정보 자동 입력
**Given** 학생이 "지원하기" 버튼을 클릭했을 때
**When** 지원서 작성 페이지가 열리면
**Then** 다음 정보가 프로필에서 자동으로 입력된다:
- 이름, 연락처, 이메일
- 학교, 전공, 학년, 졸업(예정)일
- 보유 기술 스택

**And** 학생은 다음 추가 정보를 입력할 수 있다:
- 자기소개서 (최대 1,000자)
  - 템플릿 제공: "지원 동기", "강점 및 경험", "입사 후 포부"
- 포트폴리오 링크 (선택사항)
- 이력서 파일 업로드 (PDF, 3MB 이하)

### Scenario 4: 자기소개서 AI 도우미 (Phase 2 기능)
**Given** 학생이 자기소개서를 작성 중일 때
**When** "AI 도우미" 버튼을 클릭하면
**Then** 다음 기능이 제공된다:
- 맞춤법 및 문법 검사
- 문장 개선 제안
- 공고 키워드 분석 및 반영 제안
  (예: "이 공고는 '협업'을 강조하고 있습니다. 관련 경험을 추가해보세요")

### Scenario 5: 지원서 제출 및 확인
**Given** 모든 필수 항목을 입력한 학생이
**When** "제출하기" 버튼을 클릭하면
**Then** 확인 모달이 표시된다:
- "제출하시겠습니까? 제출 후에는 수정할 수 없습니다."
- "확인" / "취소" 버튼

**When** "확인"을 클릭하면
**Then** 지원서가 제출되고:
- 성공 메시지 표시: "지원이 완료되었습니다!"
- 자동으로 "내 지원 현황" 페이지로 이동
- 기업 담당자에게 이메일 알림 전송
- 학생에게 확인 이메일 전송 (지원서 사본 포함)

### Scenario 6: 지원 현황 대시보드
**Given** 학생이 마이페이지의 "지원 현황" 탭에 접근했을 때
**Then** 다음 정보가 Kanban 보드 형식으로 표시된다:

**컬럼 구분:**
- 지원 완료 (서류 검토 중)
- 서류 합격 (면접 대기)
- 면접 진행 중
- 최종 합격
- 불합격

**각 카드에는:**
- 기업명 + 공고 제목
- 지원일
- 현재 상태
- 다음 단계 안내 (예: "면접 일정 대기 중")

**And** 상단에 KPI 요약 표시:
- 총 지원 건수: 15건
- 서류 합격률: 60% (9/15)
- 면접 진행 중: 3건
- 최종 합격: 1건

### Scenario 7: 채용 단계별 알림
**Given** 기업이 지원자의 상태를 변경했을 때
**Then** 학생에게 다음 알림이 전송된다:

**서류 합격 시:**
- 앱 푸시: "축하합니다! [기업명] 서류 전형에 합격하셨습니다."
- 이메일: 면접 안내 (일시, 장소, 준비물)
- 인앱 알림 + 지원 현황 보드 자동 업데이트

**면접 일정 확정 시:**
- 캘린더 초대 파일 (ICS) 첨부
- 면접 D-3, D-1에 리마인더 알림

**최종 합격 시:**
- 축하 메시지 + 입사 절차 안내

**불합격 시:**
- 위로 메시지 + "다른 공고 추천" 링크

### Scenario 8: 면접 일정 응답
**Given** 기업이 면접 일정을 제안했을 때
**When** 학생이 알림을 클릭하면
**Then** 면접 상세 정보가 표시되고:
- 일시, 장소/링크
- 면접 유형 (대면/화상)
- 준비물, 안내 사항

**And** 학생은 다음 액션을 선택할 수 있다:
- "참석 확정"
- "일정 조율 요청" (메시지 입력)
- "불참" (사유 선택)

**When** "참석 확정"을 클릭하면
**Then** 기업에게 알림이 전송되고, 내 캘린더에 자동 추가된다

### Scenario 9: 지원서 철회
**Given** 학생이 지원 현황 페이지에서 특정 지원 건을 선택했을 때
**When** "지원 철회" 버튼을 클릭하면
**Then** 확인 모달이 표시된다:
- "정말 지원을 철회하시겠습니까?"
- 사유 선택 (선택사항): 다른 회사 합격, 관심 변경, 기타

**When** 확인하면
**Then** 해당 지원이 취소되고:
- 기업에게 철회 알림 전송
- 지원 현황에서 제거 (또는 "철회됨" 상태로 표시)

## 인수 기준 (Acceptance Criteria)

### Functional
✅ 학생은 동일한 공고에 중복 지원할 수 없음 (에러 메시지 표시)
✅ 지원서 제출 시 프로필의 최신 정보가 반영됨
✅ 마감일이 지난 공고는 지원 불가 ("마감됨" 표시)
✅ 파일 업로드는 PDF만 허용, 3MB 이하
✅ 지원 상태 변경 시 30초 이내에 알림 전송
✅ 지원 현황 보드는 실시간으로 업데이트됨

### Non-Functional
✅ 지원서 제출 후 3초 이내에 완료 메시지 표시
✅ 지원 현황 페이지 로딩 시간 < 2초
✅ 동시 100명 지원서 제출 처리 가능

### UI/UX (Salesforce SLDS 스타일)
✅ 공고 검색 결과는 Card Grid 레이아웃
✅ 지원서 작성은 Step Indicator로 진행률 표시
✅ 지원 현황은 Kanban 보드 + Table 뷰 전환 가능
✅ 알림은 Toast + Notification Panel 표시

## 의존성
- 학생 프로필 시스템 (US-01)
- 기업 채용공고 시스템 (US-02)
- 파일 업로드 인프라 (AWS S3)
- 알림 시스템 (이메일 + 푸시)
- 캘린더 통합 (Google Calendar, Outlook)

## 기술적 고려사항
- **Frontend**: React Hook Form, File Upload Component
- **Backend**:
  - Application CRUD API
  - Status Management (FSM: Finite State Machine)
  - Notification Trigger (Event-Driven)
- **Database**:
  - Application 테이블: student_id, job_id, status, submitted_at
  - ApplicationDocument 테이블: 파일 메타데이터
- **Storage**: S3 + CloudFront (이력서 CDN)
- **Email**: SendGrid Template (상태별 다른 템플릿)

## 테스트 시나리오

### 기능 테스트
1. 마감일 지난 공고 지원 시도 → "마감된 공고입니다" 에러
2. 동일 공고 재지원 시도 → "이미 지원한 공고입니다" 에러
3. 필수 항목 누락 후 제출 → "자기소개서를 입력해주세요" 경고
4. 4MB PDF 업로드 시도 → "파일 크기는 3MB 이하여야 합니다" 에러
5. 정상 지원 완료 → 확인 이메일 수신 (30초 이내)

### 상태 전환 테스트
6. 기업이 "서류 합격" 처리 → 학생 앱에 푸시 알림 + 보드 업데이트
7. 학생이 "지원 철회" → 기업에게 알림 + 보드에서 제거

### 성능 테스트
8. 동시 100명 지원서 제출 → 모두 3초 이내 완료
9. 지원 현황 100건 로딩 → 2초 이내 표시
