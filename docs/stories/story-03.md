# US-03: AI 기반 학생-기업 매칭 및 추천

## User Story
**As a** 취업 준비 중인 대학생
**I want to** 내 프로필에 맞는 채용공고를 AI가 자동으로 추천받고
**So that** 일일이 검색하지 않아도 나에게 적합한 기회를 놓치지 않을 수 있다

## 시나리오

### Scenario 1: 로그인 후 맞춤형 채용공고 피드 표시
**Given** 학생이 플랫폼에 로그인했을 때
**When** 메인 대시보드에 접근하면
**Then** "추천 채용공고" 섹션이 상단에 표시되고:
- 매칭 점수가 높은 순서로 공고가 나열됨
- 각 공고마다 "매칭도 85%" 같은 점수 표시
- 매칭 이유 간략 표시 (예: "귀하의 기술 스택 Python, React와 일치")

**And** 최소 5개 이상의 추천 공고가 표시된다

### Scenario 2: 매칭 상세 분석 보기
**Given** 추천 공고 목록에서 특정 공고를 선택했을 때
**When** "매칭 분석 보기" 아이콘을 클릭하면
**Then** 다음 정보가 모달로 표시된다:
- 전체 매칭 점수: 85/100
- 세부 항목별 점수:
  - 기술 스택 일치도: 90% (Python, React, Node.js 일치)
  - 관심 분야 일치도: 80% (IT, 웹 개발)
  - 학력 요건 충족: 100% (학사 이상)
  - 근무지 선호도: 70% (서울 희망, 경기 공고)
  - 경력 요건: 60% (신입 가능, 1년 우대)

**And** 부족한 부분에 대한 개선 제안:
- "Node.js 프로젝트 경험을 추가하면 매칭도가 95%로 상승합니다"

### Scenario 3: 실시간 매칭 알림
**Given** 학생의 프로필이 등록된 상태에서
**When** 새로운 채용공고가 등록되고 매칭 점수가 80점 이상일 때
**Then** 다음 알림이 전송된다:
- 앱 푸시 알림: "새로운 맞춤 채용공고 1건"
- 이메일: 공고 요약 + "지금 확인하기" 링크
- 인앱 알림 배지 표시

**And** 알림 클릭 시 해당 공고 상세 페이지로 이동

### Scenario 4: 관심 기업 설정 및 우선 추천
**Given** 학생이 프로필 설정에서 "관심 기업" 메뉴에 접근했을 때
**When** 기업명을 검색하여 선택하면
**Then** 해당 기업의 새 공고가 등록될 때마다 우선 알림을 받는다

**And** 추천 피드에서 관심 기업 공고는 상단에 고정 표시되며 "⭐ 관심 기업" 배지가 붙는다

### Scenario 5: 매칭 필터 커스터마이징
**Given** 학생이 추천 피드 페이지에 있을 때
**When** "추천 설정" 버튼을 클릭하면
**Then** 다음을 조정할 수 있는 설정 패널이 표시된다:
- 최소 매칭 점수: 슬라이더로 50~100점 설정
- 고용 형태: 정규직, 인턴, 계약직 체크박스
- 경력 요건: 신입만, 경력 포함 토글
- 근무지: 지역 선택 (다중)
- 급여 범위: 최소 금액 입력

**And** 설정 저장 시 즉시 피드가 갱신된다

### Scenario 6: 기업 관점 - 추천 학생 목록
**Given** 기업이 채용공고를 발행했을 때
**When** "추천 지원자" 탭에 접근하면
**Then** 공고와 매칭도가 높은 학생 목록이 표시된다:
- 학생 이름, 학교, 전공, 학년
- 매칭 점수 (90점 이상 "적극 추천" 배지)
- 주요 일치 기술 스택 태그
- "스카우트 메시지 보내기" 버튼

**And** 기업은 학생에게 먼저 지원 제안을 보낼 수 있다

### Scenario 7: 매칭 정확도 피드백
**Given** 학생이 추천받은 공고에 지원하거나 무시했을 때
**When** "이 추천이 유용했나요?" 피드백 요청이 표시되면
**Then** 👍 (유용함) 또는 👎 (유용하지 않음) 선택 가능

**And** 👎 선택 시 이유를 묻는 옵션 제공:
- 관심 분야가 아님
- 근무지가 맞지 않음
- 경력 요건이 안 맞음
- 기타 (텍스트 입력)

**And** 이 피드백은 AI 모델 학습에 반영되어 추천 정확도를 개선한다

## 인수 기준 (Acceptance Criteria)

### Functional
✅ 매칭 알고리즘은 다음 요소를 고려해야 한다:
  - 기술 스택 일치도 (가중치 40%)
  - 관심 분야 일치도 (가중치 20%)
  - 학력/경력 요건 충족 (가중치 20%)
  - 근무지 선호도 (가중치 10%)
  - 기타 우대사항 (가중치 10%)

✅ 신규 공고 등록 후 5분 이내에 매칭 학생에게 알림 전송
✅ 학생 프로필 수정 시 추천 피드가 실시간 갱신됨
✅ 매칭 점수 계산은 1초 이내에 완료되어야 함
✅ 학생은 하루 최대 10건의 추천 알림을 받음 (스팸 방지)

### Non-Functional
✅ 추천 API 응답 시간 < 2초
✅ 동시 1,000명의 매칭 요청 처리 가능
✅ 매칭 정확도 초기 목표: 70% 이상 (학생 피드백 기반)

### UI/UX (Salesforce SLDS 스타일)
✅ 추천 공고는 Card Grid 레이아웃으로 표시
✅ 매칭 점수는 Progress Bar + 숫자로 시각화
✅ 매칭 분석 모달은 Radar Chart 또는 Bar Chart 사용
✅ 알림은 Toast 형식으로 우측 하단에 표시

## 기술적 고려사항

### Phase 1 (MVP): Rule-based 매칭
- **알고리즘**: 가중치 기반 점수 계산
  ```typescript
  matchingScore =
    (techStackMatch * 0.4) +
    (interestMatch * 0.2) +
    (educationMatch * 0.2) +
    (locationMatch * 0.1) +
    (benefitMatch * 0.1)
  ```
- **기술 스택**: PostgreSQL Full-Text Search, Redis Cache
- **실시간 알림**: Firebase Cloud Messaging, SendGrid

### Phase 2: ML 기반 고도화
- **모델**: Collaborative Filtering (학생-공고 매트릭스)
- **Feature Engineering**:
  - 학생 행동 데이터 (클릭, 지원, 거절)
  - 공고 성공률 (지원자 수, 채용 완료율)
  - 텍스트 임베딩 (공고 설명, 자기소개서)
- **ML Stack**: Python (scikit-learn, TensorFlow), MLflow
- **AB Testing**: 기존 Rule vs ML 모델 성능 비교

## 테스트 시나리오

### 기능 테스트
1. 신규 학생 가입 → 프로필 작성 → 즉시 추천 공고 5개 이상 표시
2. 기술 스택 Python만 있는 학생 → Python 요구 공고 우선 추천
3. 근무지 '서울'만 선택한 학생 → 경기 공고는 낮은 점수
4. 새 공고 등록 (기술: React) → React 보유 학생 10명에게 알림 전송

### 성능 테스트
5. 1,000개 공고 × 1,000명 학생 매칭 → 응답 시간 측정
6. 동시 100명 로그인 → 추천 피드 로딩 시간 < 2초

### 정확도 테스트
7. 실제 학생 20명 대상 A/B 테스트 → 추천 만족도 설문 (목표: 4.0/5.0)
