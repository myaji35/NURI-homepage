# US-02: 기업 채용공고 등록 및 관리

## User Story
**As a** 기업 채용 담당자
**I want to** 채용공고를 등록하고 지원자를 효율적으로 관리하고
**So that** 우리 회사에 적합한 인재를 빠르게 찾고 채용 프로세스를 간소화할 수 있다

## 시나리오

### Scenario 1: 기업 회원 가입 및 기업 정보 등록
**Given** 채용 담당자가 플랫폼에 처음 접속했을 때
**When** "기업 회원 가입" 버튼을 클릭하면
**Then** 다음 정보를 입력할 수 있는 폼이 표시된다:
- 기업명, 사업자등록번호 (인증용)
- 산업 분류, 기업 규모 (대기업/중견/중소/스타트업)
- 본사 주소, 대표 전화번호
- 기업 소개 (500자 이내)
- 기업 로고 업로드

**And** 사업자등록번호 인증 API를 통해 실제 기업임을 검증한다

### Scenario 2: 채용공고 작성
**Given** 기업 회원이 로그인한 상태에서
**When** "채용공고 등록" 버튼을 클릭하면
**Then** 다음 정보를 입력할 수 있는 작성 화면이 표시된다:

**필수 항목:**
- 공고 제목 (예: "백엔드 개발자 신입/경력 채용")
- 채용 분야: 개발, 기획, 디자인, 마케팅 등
- 고용 형태: 정규직, 인턴, 계약직
- 경력 요건: 신입, 경력 1년 이상, 경력 3년 이상 등
- 근무지
- 모집 인원
- 접수 마감일

**선택 항목:**
- 급여 정보 (연봉 범위 또는 시급)
- 요구 기술 스택 (자동완성 태그)
- 우대 사항
- 복리후생
- 채용 절차 (서류전형 → 면접 → 최종합격)

### Scenario 3: 채용공고 미리보기 및 발행
**Given** 채용공고 작성을 완료한 담당자가
**When** "미리보기" 버튼을 클릭하면
**Then** 학생이 보게 될 공고 화면이 표시되고
**And** "발행" 또는 "임시저장" 버튼을 선택할 수 있다

**When** "발행" 버튼을 클릭하면
**Then** 공고가 즉시 플랫폼에 게시되고
**And** 매칭된 학생들에게 알림이 전송된다

### Scenario 4: 지원자 목록 조회 및 필터링
**Given** 채용공고에 지원자가 있을 때
**When** "지원자 관리" 탭에 접근하면
**Then** 다음 정보가 표시된다:
- 지원자 이름, 학교, 전공, 학년
- 지원일시
- 프로필 완성도
- 매칭 점수 (AI 추천)
- 지원 상태: 서류 검토 중, 면접 대기, 합격, 불합격

**And** 다음 필터를 적용할 수 있다:
- 학교별, 전공별, 기술 스택별
- 지원 상태별
- 매칭 점수 순

### Scenario 5: 지원자 상세 프로필 확인 및 평가
**Given** 지원자 목록에서 특정 지원자를 선택했을 때
**When** 지원자 이름을 클릭하면
**Then** 상세 프로필 페이지가 모달로 열리고:
- 전체 프로필 정보 (학력, 경력, 프로젝트, 기술 스택)
- 자기소개서 (지원 시 작성한 내용)
- 첨부 파일 (이력서, 포트폴리오)

**And** 우측에 평가 패널이 표시되어:
- 별점 평가 (1~5점)
- 메모 작성 (내부용)
- 상태 변경: "서류 합격" → "면접 요청" 버튼

### Scenario 6: 면접 일정 제안
**Given** 특정 지원자를 서류 합격 처리했을 때
**When** "면접 일정 보내기" 버튼을 클릭하면
**Then** 다음을 입력할 수 있는 폼이 표시된다:
- 면접 일시 (날짜 + 시간)
- 면접 장소 (주소 또는 "화상 면접")
- 화상 면접인 경우 Zoom/Google Meet 링크
- 안내 사항 (준비물, 복장 등)

**And** 학생에게 이메일 + 앱 푸시 알림이 전송된다

### Scenario 7: 채용공고 통계 대시보드
**Given** 기업 담당자가 메인 대시보드에 접근했을 때
**When** "채용 현황" 탭을 클릭하면
**Then** 다음 정보가 Card 형식으로 표시된다:
- 진행 중인 공고 수
- 총 지원자 수
- 평균 매칭 점수
- 공고별 조회수 및 지원율

**And** 차트로 시각화:
- 주간/월간 지원자 추이 그래프
- 학교별/전공별 지원자 분포 파이 차트

## 인수 기준 (Acceptance Criteria)

### Functional
✅ 사업자등록번호 인증이 성공해야만 기업 회원가입이 완료된다
✅ 채용공고 필수 항목이 모두 입력되어야 발행 가능하다
✅ 지원자는 공고 발행 즉시 검색 결과에 노출된다
✅ 매칭 점수는 학생 프로필과 공고 요구사항을 비교하여 자동 계산된다
✅ 기업은 자신이 등록한 공고만 수정/삭제할 수 있다
✅ 지원자 상태 변경 시 학생에게 실시간 알림이 전송된다

### Non-Functional
✅ 지원자 목록 로딩 시간은 2초 이내여야 한다
✅ 공고 발행 후 1분 이내에 검색 인덱스에 반영되어야 한다
✅ 동시 접속자 100명 이상 처리 가능해야 한다

### UI/UX (Salesforce SLDS 스타일)
✅ 채용공고 작성은 Step-by-Step 위저드 형식으로 구성
✅ 지원자 관리는 Table + Kanban 보드 뷰 제공
✅ 대시보드는 KPI Card + 차트로 구성
✅ 모든 Action(발행, 상태 변경)은 확인 모달 표시

## 의존성

## 기술적 고려사항
- **Frontend**: React Table Library (TanStack Table), Chart.js
- **Backend**: Job Posting CRUD API, Applicant Management API
- **Database**: Company, JobPosting, Application 테이블
- **Matching Algorithm**: 기술 스택 일치도 + 학년 + 희망 직무 가중치 계산
- **Notification**: SendGrid (이메일), Firebase Cloud Messaging (푸시)

## 테스트 시나리오
1. 유효하지 않은 사업자등록번호 입력 → 인증 실패 메시지
2. 필수 항목 누락 후 발행 시도 → 경고 및 저장 차단
3. 공고 발행 후 매칭 학생 100명에게 알림 전송 → 1분 이내 완료
4. 지원자 상태를 "서류 합격"으로 변경 → 학생에게 알림 도착 확인
5. 대시보드에서 최근 30일 지원자 추이 그래프 정상 표시
