# US-01: 학생 프로필 생성 및 관리

## User Story
**As a** 취업 준비 중인 대학생
**I want to** 나의 학력, 경력, 기술 스택, 관심사를 체계적으로 등록하고 관리하고
**So that** 기업 채용 담당자가 나의 역량을 정확하게 파악하고, 맞춤형 채용 공고를 추천받을 수 있다

## 시나리오

### Scenario 1: 신규 회원가입 및 기본 프로필 작성
**Given** 사용자가 처음 플랫폼에 접속했을 때
**When** 이메일 인증 후 회원가입을 완료하고
**Then** 다음 정보를 입력할 수 있는 프로필 작성 화면이 표시된다:
- 기본 정보: 이름, 연락처, 생년월일
- 학력 정보: 대학명, 전공, 학년, 졸업(예정)일
- 프로필 사진 업로드

### Scenario 2: 상세 프로필 작성 (기술 스택 및 관심사)
**Given** 기본 프로필을 작성한 사용자가
**When** "프로필 상세 정보 입력하기" 버튼을 클릭하면
**Then** 다음 항목을 추가로 입력할 수 있다:
- 기술 스택: 프로그래밍 언어, 프레임워크, 도구 (다중 선택 + 검색)
- 관심 분야: IT, 금융, 제조, 마케팅 등 (태그 형식)
- 희망 직무: 개발자, 기획자, 디자이너 등
- 희망 근무지: 지역 선택 (다중 가능)

### Scenario 3: 경력 및 프로젝트 이력 추가
**Given** 사용자가 프로필 편집 모드에 있을 때
**When** "경력 추가" 또는 "프로젝트 추가" 버튼을 클릭하면
**Then** 다음 정보를 입력할 수 있는 모달이 열린다:

**경력 항목:**
- 회사명, 직무, 근무 기간, 업무 내용

**프로젝트 항목:**
- 프로젝트명, 역할, 기간, 사용 기술, 설명, 링크 (GitHub, 포트폴리오 등)

### Scenario 4: 자격증 및 수상 경력 등록
**Given** 사용자가 "자격증/수상" 섹션에 있을 때
**When** "추가" 버튼을 클릭하면
**Then** 자격증명, 발급기관, 취득일 또는 수상명, 주최기관, 수상일을 입력할 수 있다

### Scenario 5: 프로필 공개 설정 관리
**Given** 사용자가 프로필을 작성 완료했을 때
**When** "공개 설정" 메뉴에 접근하면
**Then** 다음을 선택할 수 있다:
- 전체 공개: 모든 기업이 프로필 조회 가능
- 제한 공개: 내가 지원한 기업만 조회 가능
- 비공개: 아무도 조회 불가 (검색 제외)

**And** 각 항목별(연락처, 학교 정보 등) 개별 공개/비공개 설정 가능

### Scenario 6: 프로필 완성도 시각화
**Given** 사용자가 프로필 페이지를 볼 때
**When** 페이지 상단에 프로필 완성도가 표시되고
**Then** 다음과 같이 표시된다:
- 진행률 바: "프로필 완성도 75%"
- 미입력 항목 가이드: "기술 스택을 추가하면 +15%"

## 인수 기준 (Acceptance Criteria)

### Functional
✅ 사용자는 이메일 인증을 통해 회원가입할 수 있어야 한다
✅ 기본 정보(이름, 학교, 전공, 학년)는 필수 입력이어야 한다
✅ 기술 스택은 자동완성 검색으로 선택할 수 있어야 한다
✅ 프로젝트 링크는 URL 유효성 검증이 되어야 한다
✅ 프로필 사진은 5MB 이하, JPG/PNG 형식만 업로드 가능해야 한다
✅ 모든 입력 데이터는 저장 시 자동으로 DB에 반영되어야 한다
✅ 프로필 완성도는 실시간으로 계산되어야 한다

### Non-Functional
✅ 프로필 저장은 3초 이내에 완료되어야 한다
✅ 모바일에서도 모든 기능이 정상 작동해야 한다
✅ WCAG 2.1 AA 수준의 접근성을 준수해야 한다

### UI/UX (Salesforce SLDS 스타일)
✅ Card 기반 레이아웃으로 섹션별 구분
✅ 상단에 프로필 완성도 KPI 표시
✅ 우측 상단에 "저장", "미리보기" Action 버튼 배치
✅ 입력 필드는 Label + Helper Text 구조로 명확성 확보

## 의존성
- 인증 시스템 구현 필요
- 파일 업로드 시스템 (AWS S3 또는 Cloudinary)
- 학교 목록 DB 구축

## 기술적 고려사항
- **Frontend**: React + TypeScript, Form Validation (Zod/Yup)
- **Backend**: REST API, Multipart File Upload
- **Database**: User, Profile, Experience, Project 테이블 설계
- **Storage**: 이미지 CDN 연동

## 테스트 시나리오
1. 유효하지 않은 이메일 형식으로 회원가입 시도 → 오류 메시지 표시
2. 필수 항목 미입력 후 저장 시도 → 경고 메시지 및 저장 차단
3. 5MB 초과 이미지 업로드 시도 → 파일 크기 오류 표시
4. 프로필 완성도 0%에서 모든 항목 입력 → 100% 달성
